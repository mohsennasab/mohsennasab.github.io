# hydromohsen.com

Personal website for [Dr. Mohsen Tahmasebi Nasab](https://www.hydromohsen.com/), built with
**[Astro](https://astro.build)**, to replace the current Google Sites site and be hosted free on
**GitHub Pages** with the `hydromohsen.com` domain pointed at it (DNS stays at Squarespace).

Design: dark theme by default (light toggle included), typeface **Hubot Sans**, "Stormy Morning"
color palette (`#6A89A7` `#BDDDFC` `#88BDF2` `#384959`).

Nine pages, all with real content: About (home), Publications (from Google Scholar), Projects
(consulting and project work, client names withheld), Tutorials (the *Python Fundamentals for H&H*
course plus YouTube playlists), Photography (gallery + lightbox), Tools & Apps (from GitHub),
Teaching (philosophy, courses, lecture playlists, student feedback, a senior-design video), Blog,
and Media. Blog post text is still being moved over from the previous site.

## Prerequisites

Node.js 18.17+ is required. A portable Node 24 LTS is installed at
`%USERPROFILE%\nodejs\node-v24.15.0-win-x64` and is on your User PATH, so a new terminal already has
`node` and `npm`. To remove it later, delete that folder and the matching `Path` entry in *Edit
environment variables for your account*, or install the regular package from <https://nodejs.org>.

## Run it locally

```powershell
cd "c:\OneDrive\OneDrive - AECOM\Automation Projects\Website"
npm install        # first time only; already done
npm run dev        # dev server at http://localhost:4321  (edits hot-reload, Ctrl+C to stop)
```

| Command           | What it does                                   |
| ----------------- | ---------------------------------------------- |
| `npm run dev`     | Local dev server with hot reload               |
| `npm run build`   | Production build into `dist/`                   |
| `npm run preview` | Serve the built `dist/` locally to sanity-check |

## Project layout

```
.
├── astro.config.mjs        # site URL and integrations
├── public/                 # served verbatim at the site root
│   ├── CNAME               # custom domain for GitHub Pages (hydromohsen.com)
│   ├── favicon.svg
│   └── robots.txt
│   (later) public/apps/<name>/   <- drop a standalone HTML app here, served at /apps/<name>/
├── .github/workflows/deploy.yml   # builds and publishes to GitHub Pages on push to main
└── src/
    ├── consts.ts            # site name, nav items, social links, GitHub username
    ├── styles/global.css    # the whole design system (CSS variables, dark/light)
    ├── layouts/Base.astro   # <head>, fonts, nav, footer, theme toggle, view transitions
    ├── components/          # Nav, Footer, Hero, Socials, Card, SectionHead, PageHeader
    ├── assets/              # headshot, portrait, podcast photo, and photos/ (39 gallery images)
    └── pages/               # one file = one route
        ├── index.astro                       -> /            (About / Home)
        ├── publications.astro                -> /publications
        ├── projects.astro                    -> /projects
        ├── tutorials/index.astro             -> /tutorials   (overview)
        ├── tutorials/python-fundamentals.astro -> /tutorials/python-fundamentals
        ├── photography.astro                 -> /photography
        ├── tools.astro                       -> /tools
        ├── teaching.astro                    -> /teaching
        ├── blog/index.astro                  -> /blog
        ├── media.astro                       -> /media
        └── 404.astro                         -> custom not-found page
```

Note: the original full-resolution photo folders (`Mohsen/`, `Photos/`) are git-ignored. The
web-sized copies used by the site live in `src/assets/` (and `src/assets/photos/`), generated by
`_convert_images.py` (also git-ignored). Re-run that script if you add or change source photos.

### Common edits

- Site name, nav, social links, GitHub username: `src/consts.ts`
- Colors, fonts, spacing: the `:root` and `[data-theme="light"]` blocks in `src/styles/global.css`
- Add a page: create `src/pages/whatever.astro` (becomes `/whatever`) and add it to `NAV` in `consts.ts`
- Embed your own HTML app: put it in `public/apps/<name>/`, live at `/apps/<name>/`, link or `<iframe>` it from any page
- Add/replace photos: drop new images in `Photos/`, run `_convert_images.py`, the gallery picks them up automatically

## Deploying (later)

1. Push this folder to a GitHub repo (recommended name: `<your-username>.github.io`).
2. Repo **Settings, Pages, Source = GitHub Actions**. The included workflow builds and publishes on every push to `main`.
3. Repo **Settings, Pages, Custom domain = `hydromohsen.com`** (the `public/CNAME` file already has it).
4. In the **Squarespace DNS** panel for `hydromohsen.com`, add:
   - **A** records for `@`: `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - **AAAA** records for `@`: `2606:50c0:8000::153`, `2606:50c0:8001::153`, `2606:50c0:8002::153`, `2606:50c0:8003::153`
   - **CNAME** for `www`: `<your-username>.github.io`
   - remove the old Google Sites records (the `ghs.googlehosted.com` CNAME, etc.)
5. After DNS propagates, tick **Enforce HTTPS** in Settings, Pages.
